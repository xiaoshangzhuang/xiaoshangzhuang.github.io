在linux系统部署web项目

在linux系统上部署web项目
一．在linux系统上安装tomcat
1.下载tomcat
1.1进入 Apache Tomcat官网，选择要下载的版本。
1.2把tomcat安装包上传到服务器 /usr/local 下。

2.解压安装tomcat
2.1用Xshell等Linux远程连接工具访问服务器，进入文件夹，指令是cd /usr/local。
2.2解压安装包，指令是unzip apache-tomcat-7.0.78.tar.gz。
2.3进入bin目录，指令是cd /usr/local/tapache-omcat-7-0-78/bin。
2.4启动tomcat，指令是sh startup.sh。
2.5在ogs下查看启动日志，指令是tail -f catalina.out，可以查看到tomcat的运行信息，如果看到INFO: Server startup in 1925 ms，表示tomcat已经启动成功了，按ctrl+c退出该日志。
2.6在浏览器输入服务器的IP+端口（默认8080），能看到如下页面，这是tomcat没有部署项目的默认页面。

2.7停止运行tomcat，指令是sh shutdown.sh。

二．在linux系统上搭建java环境
1.在Centos下使用yum安装java环境。
2.查看yum库的java安装包，指令是 yum -y list java*。

3.选择合适的版本，使用yum安装。

4.查看结果，指令是 java -version，如果有java版本就表示安装成功。

三．在linux系统上安装nginx
1.下载nginx
1.1进入nginx官网，选择要下载的版本。
1.2把nginx安装包上传到服务器 /usr/local 目录下。
2.解压安装nginx
2.1解压安装之前，先检查是否安装pcre，如果没有，就用yum 安 装。命令是，yum -y install pcre 和 yum -y install pcre-devel。找 到 pcre位置，命令是 rpm -ql pcre 和 rpm -ql pcre-devel。
2.2解压nginx，命令是tar -zxvf nginx-1.10.2.tar.gz，进入 目 录，命令是，cd/usr/local/nginx-1-10-2，执行命令 ./configure。

如果这一步出错了，根据错误提示信息解决问题后，使用 make clean 命令，清除产生的临时文件，再来重新执行 ./configure
2.3编译和安装
执行命令，make 和 make install，到此安装完毕。
2.4修改nginx配置文件，命令是，vi /conf/nginx.conf。修改端口为80。
2.5启动nginx，命令是，./nginx -c。在浏览器里输入服务器ip或域名，如下图，表示成功。

Hexo个人微博搭建

Hexo搭建分为以下几个步骤
1.搭建环境准备（包括node.js和git环境，gitHub账户的配置）
2.安装Hexo
3.配置Hexo
4.申请GitHub账号
5.申请外网域名（可用）
6.本地hexo与GitHub pages绑定

1.1.1：
Node.js环境搭建步骤：
1）：下载Node.js安装文件：安装包分为32位与64位
2）：安装界面

3）：打开CMD输入如下命名确认是否安装成功Node.js
1：node -v
2:npm -v

1:Git环境搭建
Git环境搭建步骤：
1）：下载Git安装包
2）：运行安装包如下

Git安装界面

Git PATH设置

3):检查是否安装成功,输入git –service如图则表示安装成功

1.1.3:
gitHub账号申请,环境搭建：
1): 输入https://github.com/ 申请gitHub账号
2)：登陆之后，创建代码库，点击页面右上角的加号，选择New repository

3）进入代码库创建页面:在Repository name下填写 youname.github.io,Description(optional)下填写一些简单的 描述（不写也没关系）,如图所示

注意:比如github名称是gdutxiaoxu,这里就填写 gdutxiaoxu.github.io.
4):代码库设置
接下来开启gh-pages功能，点击界面右侧的Settings，你将会打开这个库的setting页面，向下拖动，直到看见GitHub Pages，如图：

点击Automatic page generator，Github将会自动替你创建出一个gh-pages的页面。 如果你的配置没有问题，那么大约15分钟之后，yourname.github.io这个网址就可以正常访问了~ 如果yourname.github.io已经可以正常访问了，那么Github一侧的配置已经全部结束了。
到此搭建hexo博客的相关环境配置已经完成，下面开始讲解Hexo的相关配置
2： Hexo安装
2.1.1：在自己认为合适的地方创建一个文件夹，这里我以E：/hexo 为例子讲解， 首先在E盘目录下创建Hexo文件夹，并在命令行的窗口进入到该目录

在命令行中输入npm install hexo-cli -g

然后你将会看到如下:

可能你会看到一个WARN，但是不用担心，这不会影响你的正常使用。 然后输入：npm install hexo –save
然后你会看到命令行窗口刷了一大堆白字，下面我们来看一看Hexo是不是已经安装好了。 在命令行中输入： hexo -v 如果你看到了如图文字，则说明已经安装成功了。

Hexo的相关配置
1：初始化hexo,接着上面的操作输入：hexo init
然后接着输入npm install之后npm将会自动安装你需要的组件，只需要等待npm操作即可。
2：首次体验Hexo继续输入命令hexo g显示如下数据

然后在输入：hexo s然后显示如下信息可访问本地微博
INFO Hexo is running at http://0.0.0.0:4000/. Press Ctrl+C to stop.
在浏览器中打开http://localhost:4000/，你将会看到：

3：将Hexo与gitHub page关联
大概分为以下几步 

- 配置git个人信息 
- 配置Deployment

1) 配置Git个人信息： 如果你之前已经配置好git个人信息，请跳过这一个 步骤，直接来到

  1. 设置Git的user name和email：(如果是第一次的话)
    git config –global user.name “cai”
    git config –global user.email “987903100@qq.com”
    1. 生成秘钥
      ssh-keygen -t rsa-C “987903100@qq.com”
      2) 配置Deployment
      找到 _config.yml添加如下代码：
      deploy:
      type: git
      repo: git@github.com:shazicai/shazicai.github.io.git
      branch: master
      备注：repo配置与仓库名想对应shazicai改为自己的仓库名称

3) 登录github，找到项目，设置setting，找到GitHub Pages

4): 在master分支下的CNAME.MD文件写入注册的域名

输入绑定域名就可访问本地提交微博页面
